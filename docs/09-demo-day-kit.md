# Demo Day Kit

Demo day makes the work public, but it should not turn every team into a stage act. All teams build and get feedback. Finalists pitch publicly.

## Run of Show

- 00:00 Doors, project gallery, informal demos.
- 00:20 MC welcome, format, code of conduct, judging reminder.
- 00:30 Finalist demos begin.
- 01:30 Break and judge scoring.
- 01:45 Non-finalist lightning gallery or public browsing.
- 02:05 Awards and acknowledgments.
- 02:25 Continuation asks and next 30 days.
- 02:35 Group photo and open demos.

## Moderator Script Skeleton

Welcome to AI-Native Citizens: [City]. This room is here for working civic prototypes, not slideware.

Every team used the same global rubric. Every team must publish a build log. The build log matters because the public needs to see how these tools were made, where AI helped, and where humans corrected it.

Each finalist has 5 minutes to demo and 3 minutes for questions. Judges will score the work, the process, the civic usefulness, the clarity, and the next step.

## Finalist Selection

Mentors nominate teams based on demo readiness, civic usefulness, and build-log completeness. Organizers select a balanced set across challenge categories. Judges should not change finalist selection after public demos begin.

## Slide Template Guidance

Slides are optional. If used, keep them to:

- Problem and named user.
- What the prototype does.
- Live demo link or QR code.
- What works and what does not.
- Next 30-day ask.

No fake traction metrics. No generic AI graphics.

## Demo Rules

Time limit: 5 minutes demo, 3 minutes questions.

Working means a judge or participant can see the main user path run live, or inspect a recorded fallback only if live access is blocked by venue, login, or public data downtime.

Teams must state limits before questions.

## Awards Script

This award recognizes a team that used AI-native workflows with strong human judgment and made a civic problem more understandable or usable.

The prototype is not production software. It is a public starting point. The next step is follow-through, credit, and support.
