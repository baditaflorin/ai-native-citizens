# Contributing

AI-Native Citizens accepts practical contributions: clearer docs, better challenge briefs, workflow cards tested in the field, translations, templates, and build logs from real civic AI sprints.

Repo: https://github.com/baditaflorin/ai-native-citizens

## Ways to Contribute

- Host a Community Edition sprint and publish the build logs.
- Improve a doc in `docs/`.
- Add a challenge using the structure in `docs/04-challenge-bank.md`.
- Add a workflow card using the structure in `docs/05-ai-workflow-cards.md`.
- Translate the manifesto or templates.
- Submit a build log using `templates/ai-build-log.md`.

## Contribution Rules

- Use "AI-Native Citizens" with the hyphen on first reference.
- Keep writing direct, civic, and plain.
- Avoid corporate filler language.
- Keep files small and single-purpose.
- Prefer Markdown that works in a terminal.
- Use Conventional Commits.
- Do not add paid services, analytics, trackers, or third-party scripts.
- Do not add secrets to the repository.

## Running a Community Edition

You may use the kit without approval if you:

- Use the name "AI-Native Citizens community edition".
- Use the global judging rubric verbatim.
- Publish projects and build logs.
- Respect the code of conduct.
- Make no production-software promises.

## Submitting a Build Log

Every sprint team should publish a build log with:

- Problem
- User
- Tools used
- 5 key prompts
- What AI generated well
- What was wrong and corrected manually
- What was hard
- What works
- What does not work
- Next steps
- Repo or demo link

Use `templates/ai-build-log.md`.

## Conventional Commit Examples

```text
docs: add school rights assistant challenge
docs: clarify mentor feedback structure
feat: add static landing page starter scaffold
fix: improve mobile focus styles
```
